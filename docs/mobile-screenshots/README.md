# Mobile App Screenshots — Play Console assets

Pre-sized listing assets for the Client St0r mobile app. Every PNG
matches a Play Console slot's exact aspect ratio (16:9 or 9:16) and
size range, so they upload without resizing.

## Feature graphic

`feature-graphic.png` — **1024 × 500** PNG, the hero image at the top
of the Play Store listing. Generated by `scripts/generate_feature_graphic.py`.

## Screenshots

Each folder below holds the same 10 screens (login, dashboard, tickets,
assets, vault, kb, vehicles, inventory, organizations, settings) sized
for one Play Console slot:

| Folder        | Dimensions      | Ratio | Play Console slot                  |
|---------------|-----------------|-------|------------------------------------|
| `phone/`      | 1080 × 1920     | 9:16  | Phone (≥1080 for promotion)         |
| `tablet-7/`   | 1215 × 2160     | 9:16  | 7-inch tablet                       |
| `tablet-10/`  | 1620 × 2880     | 9:16  | 10-inch tablet (≥1080 each side)    |
| `chromebook/` | 1920 × 1080     | 16:9  | Chromebook (≥1080 each side)        |
| `android-xr/` | 1920 × 1080     | 16:9  | Android XR (≥720 each side)         |

Phone / 7"-tab / 10"-tab are portrait; Chromebook / XR are landscape
(those targets normally run in landscape, and the app supports
auto-rotation as of v3.17.475).

## How they were generated

`scripts/generate_mobile_screenshots.py` drives a headless Chromium
against the Expo web export of the mobile bundle:

1. `npx expo export --platform web` → static bundle to `/tmp/mobile-web-export/`
2. SPA-fallback HTTP server on :8765 serves the bundle (always returns
   `index.html` for unknown paths so expo-router history-mode routes
   resolve client-side).
3. Selenium with CDP forces the exact viewport via
   `Emulation.setDeviceMetricsOverride` so the PNG lands on a precise
   16:9 / 9:16 size (without this, headless Chromium shrinks the
   viewport for browser chrome and the aspect ratio comes out wrong).
4. Login is done through the real `/auth/login/` endpoint with the
   `screenshot` superuser, which has Membership rows on all 6 demo
   orgs — so populated data-laden screens (dashboard counts, assets
   list, tickets list) render instead of empty-states.

```sh
cd mobile && npm install
cd .. && /home/administrator/venv/bin/python scripts/generate_mobile_screenshots.py
/home/administrator/venv/bin/python scripts/generate_feature_graphic.py
```
